# Virtual DOM은 어떻게 작동하는가

<br />
React를 사용하는 프론트엔드 개발자라면 Virtual DOM과 뗼레야 뗄 수 없는 관계입니다.
하지만 우리는 Virtual DOM에 대해 전반적인 개념만 알 뿐 어떻게 작동하는지, React는
Virtual DOM을 왜 사용하는지 등 깊이 있는 탐구를 해본적은 많지 않을 것 입니다. 최근에
글을 읽던 중 Virtual DOM에 대해 분석한 좋은 글이 있어서 이렇게 소개를 해봅니다.

<br />
<br />
#### Introduction

<br />
React를 배우는 과정에서 Virtual DOM이란 단어를 피해갈 순 없을것이다 (아마도 영원히).
이 글은 Virtual DOM이 어떻게 React에서 실제로 구현되는 지를 명확히 하고, 추후 작성하게
될 Virtual DOM의 여러 특성과 동작들 (elements를 삭제하거나 state와 props를 업데이트
하는 등)의 초석을 닦는것을 목표로 한다.

<br />이 글을 읽기에 앞서, 나는 정중하게 여러분께 이 질문을 던진다. 여러분은
Virtual DOM이 React에서 어떻게 구현된다고 생각하십니까? 여러분은 아마 이 주제에
대해 수많은 글과 포스팅들을 읽었을 것이므로 지금까지 습득한 지식을 바탕으로 실제
구현에 대해 생각하는 것이 이 글을 최대한 활용하는 데 도움이 될 것입니다.

<br />몇 가지 전제조건은 React와 익숙하고 Javascript에 대한 적절한 예시일
것이다. 그럼 포스팅을 시작해보자.

<br />
<br />
#### About the virtual DOM

<br />
아시다 시피 DOM(Document Object Model)은 웹 페이지를 표현하기 위한 트리 구조이다.
이러한 DOM들은 데이터를 불러와 표현하거나 웹페이지의 컨텐츠의 구조를 조작하려는 개발자들에게
여러 인터페이스를 제공한다. 하지만 DOM을 직접적으로 조작하는 행위는 비효율적이고
비생산적이므로 자주 일어나서는 안된다. <code>offsetHeight</code>과 같은 특정
엘리먼트의 속성에 접근할 때도 있지만 결과적으로 브라우저를 리렌더링 시키고
비효율적인 연산을 초래하게 된다.

<br />
따라서 DOM에 대한 작업들은 꼭 필요할 때 효율적으로 수행되어야 하고, 바로 이럴 때
Virtual DOM이 빛을 발휘하게 된다. 또한 Virtual DOM은 메모리에 저장되며 페이지의 요소들을
저장하는 트리와 같은 구조지만, 어떤 작업을 수행하든 그에 비해 저렴하고 빠르다는 장점이
있다. 요소들의 트리구조 측면에서 무엇이 변경되었는지 확인한 후 모든 수정 사항이 실제
DOM에서 한 번에 커밋되는 책임을 지는 것이 바로 가상 DOM이다.

<br />
Virtual DOM을 한마디로 표현하자면 <code>FiberNode</code>들로 이루어진 트리
구조이다.
